How does BREACH work?
The BREACH attack is a compression side-channel attack, which targets information compressed in HTTP response bodies. This attack can be used to extract login credentials, anti-CSRF tokens and other sensitive, personally identifiable information from SSL-enabled websites.
Deflate (stylized as DEFLATE) is a lossless data compression file format that uses a combination of LZ77 and Huffman coding.
LZ77 replaces occurrences of three or more characters with "pointer" values to reduce space.
Huffman coding replaces characters with symbols in order to optimize the description of the data to the smallest size possible.
Breakthrough point ï¼šThe length of the compressed data is still visible after compressing with the DEFLATE algorithm.
Mechanism:
1. Injecting guesses of the secrets into the HTTP response bodies.
2. Observing the time when responses would be highly compressed, and the output length differs, means the guess matches.
3. Detecting the secret information.
4. Extracting the complete secret.

How to prevent?

Length Hiding

Useless value added into compressed response or confusion of the autual length will impair attacker's ability to calculate the differences in length.

Separating Secrets from User Input

Instead of providing secrets in the header, serve it in a different context, which will completely prevent attack that achieves via varying input.

Masking Secrets

Mask the secret with a random disposable value to produce a new secret every time.

Request Rate-Limiting and Monitoring

Implementation of the attack needs to send thousands of requests in a short time, which is impossible for a human. Detecting it can help to defense from the potential attacks.
